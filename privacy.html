Privacy Policy

Last Updated: March 18, 2026

DriveNest ("application", "we"), is an open-source cloud storage client that places extreme importance on user privacy and data security. This document explains what data is processed and how it is protected when you use the desktop application.

1. Data Collected and Processed

DriveNest establishes a connection directly between Google APIs and your local device. Your data never goes to a server hosted by us.

  • Google Account Data (OAuth2): Requested so you can log in through the application and see your profile email address.
  • Google Drive Files: Used for core operations such as listing, downloading, uploading, or deleting (moving to trash) your files. Your content is NEVER shared with an external AI or telemetry service.

2. Data Storage and Encryption

The architecture we developed is based on the "Privacy First" principle:

  • Access Tokens: Your Google API tokens are stored encrypted with hardware support in Linux "libsecret" (keyring via Keytar), Windows "Credential Vault", or macOS "Keychain".
  • Local Cache (SQLite): The SQLite database we use for fast loading of your file tree structure only holds metadata from the current session. All session information is destroyed the moment you log out.

3. Third Party Sharing

Under no circumstances will your personal data, file history, or identity information be shared, leased, or sold to third-party legal entities/companies, advertising networks, or data collectors. Our open-source code is proof of this guarantee.

4. User Control

You can permanently delete all permissions you have granted for the application to function with a single click at any time via your official Google Account's "Third-party apps with account access" panel.

5. Contact Us

If you have any doubts or suggestions regarding this policy or our data security approaches, you can contact us at any time by creating an "Issue" through our GitHub Repository ("DriveNest").
